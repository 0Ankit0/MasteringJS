JavaScript Event Loop

Introduction

JavaScript is a single-threaded language that handles asynchronous operations using the Event Loop. This mechanism allows JavaScript to process multiple tasks efficiently without blocking execution.

Understanding the Event Loop

The event loop ensures that JavaScript executes tasks in an organized manner. It follows these steps:

  1. Execute synchronous code (everything inside the main thread).
  2. Process all microtasks (e.g., Promises, MutationObserver).
  3. Render updates if necessary.
  4. Process the next macrotask (e.g., setTimeout, setInterval).

Macrotasks vs. Microtasks

Macrotasks

  • setTimeout
  • setInterval
  • setImmediate (Node.js)
  • I/O operations

Microtasks

  • Promises (.then(), .catch())
  • MutationObserver
  • queueMicrotask()

Code Example

Consider the following JavaScript code that demonstrates how the event loop processes tasks: 

console.log('Script start');

setTimeout(() => {
    console.log('setTimeout (Macrotask)');
}, 0);

Promise.resolve().then(() => {
    console.log('Promise 1 (Microtask)');
}).then(() => {
    console.log('Promise 2 (Microtask)');
});

console.log('Script end');

Expected Output

Script start
Script end
Promise 1 (Microtask)
Promise 2 (Microtask)
setTimeout (Macrotask)

Execution Order Explanation

The execution happens in the following order:

  1. Synchronous code executes first: 
    Script start
Script end
  2. Microtasks execute next: 
    Promise 1 (Microtask)
Promise 2 (Microtask)
  3. Macrotasks execute last: 
    setTimeout (Macrotask)
