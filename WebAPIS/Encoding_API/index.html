Understanding the Encoding API

The Encoding API allows developers to work with different text encodings such as UTF-8 and Base64. It plays a vital role in handling multilingual text, encoding binary data like images, and ensuring compatibility between systems.

Key Use Cases

  • Handling text in various encodings (e.g., UTF-8, UTF-16).
  • Encoding/decoding binary data (e.g., images) to Base64.
  • Ensuring compatibility with internationalized text.
  • Preventing security vulnerabilities like XSS.

Code Examples

1. Encoding and Decoding Base64


// Encoding a string to Base64
const text = "Hello, Encoding API!";
const encodedText = btoa(text); // Encode to Base64
console.log(encodedText); // Output: "SGVsbG8sIEVuY29kaW5nIEFQIQ=="

// Decoding Base64 back to a string
const decodedText = atob(encodedText);
console.log(decodedText); // Output: "Hello, Encoding API!"

2. Encoding an Image to Base64


// Convert an image to Base64 using FileReader
const fileInput = document.getElementById("image-input");
fileInput.addEventListener("change", () => {
  const file = fileInput.files[0];
  if (file) {
    const reader = new FileReader();
    reader.onload = (event) => {
      const base64Image = event.target.result;
      console.log(base64Image); // Data URI with Base64-encoded image
    };
    reader.readAsDataURL(file);
  }
});

Real-World Applications

  • Chat Apps: Encode images for secure transmission.
  • Email Clients: Attachments are Base64-encoded for transport.
  • QR Code Generators: Encode data for QR code content.
  • Web APIs: Encode binary data for API compatibility.
