Navigation API - Complete Guide

The Navigation API enables better control over browser navigation history and transitions.

1. navigation.navigate(url, options)

Purpose: Navigates to a new URL programmatically.

Parameters:

  • url (string) – The destination URL (absolute or relative).
  • options (object, optional) – Custom options:
    • state: A custom object that gets stored with this navigation.
    • replace: If true, replaces current history entry instead of creating a new one.

Example:

navigation.navigate('/dashboard', { state: { userId: 42 }, replace: false });

2. navigation.currentEntry

Purpose: Retrieves information about the current navigation entry.

Properties:

  • url: The current URL.
  • state: The state object passed during navigation.
  • key: A unique identifier for this history entry.

Example:

console.log(navigation.currentEntry.url);

3. navigation.entries()

Purpose: Retrieves the entire navigation history stack.

Example:

const entries = navigation.entries();
console.log(entries);

4. navigation.canGoBack / navigation.canGoForward

Purpose: Checks if the user can go back or forward in history.

Example:

if (navigation.canGoBack) console.log('Can go back');
if (navigation.canGoForward) console.log('Can go forward');

5. navigation.back() / navigation.forward()

Purpose: Moves the user backward or forward in the history stack.

Example:

navigation.back(); // Go back
navigation.forward(); // Go forward

6. navigation.traverseTo(key)

Purpose: Navigates to a specific history entry.

Example:

const firstEntry = navigation.entries()[0];
if (firstEntry) navigation.traverseTo(firstEntry.key);

7. navigation.reload()

Purpose: Reloads the current page without losing state.

Example:

navigation.reload();

8. Limitations

  • ⚠️ Limited browser support (Chrome, Edge, but not Safari or Firefox).
  • ⚠️ Requires JavaScript-based routing in SPAs.
  • ⚠️ Not backward-compatible with older browsers.
