Application MIME Types

Back to Overview

1. application/json

What it represents: JSON (JavaScript Object Notation), a lightweight data-interchange format.

Use case: Commonly used in APIs for data exchange between the server and client in a structured, human-readable format. 

Content-Type: application/json

Example:

{
  "name": "John",
  "age": 30
}

2. application/xml

What it represents: XML (eXtensible Markup Language), a markup language that defines a set of rules for encoding documents.

Use case: Used for representing structured data in a format that is both human-readable and machine-readable. 

Content-Type: application/xml

Example:

<person>
  <name>John</name>
  <age>30</age>
</person>

3. application/pdf

What it represents: PDF (Portable Document Format), a file format developed by Adobe to present documents.

Use case: Used for documents that need to be printed or shared in a fixed layout format. 

Content-Type: application/pdf

Example:

Content-Type: application/pdf

4. application/zip

What it represents: ZIP, an archive file format that supports lossless data compression.

Use case: Used for compressing files to reduce their size for storage or transmission. 

Content-Type: application/zip

Example:

Content-Type: application/zip

5. application/octet-stream

What it represents: Arbitrary binary data.

Use case: Used for files that do not fit into any other MIME type. 

Content-Type: application/octet-stream

Example:

Content-Type: application/octet-stream

6. application/x-www-form-urlencoded

What it represents: Form data encoded as key-value pairs.

Use case: Used for submitting form data in HTTP requests. 

Content-Type: application/x-www-form-urlencoded

Example:

Content-Type: application/x-www-form-urlencoded

Use case: Used for spreadsheets created by Microsoft Excel.

Example:

Content-Type: application/vnd.ms-excel

Content-Type: application/vnd.openxmlformats-officedocument.spreadsheetml.sheet

Content-Type: application/msword
