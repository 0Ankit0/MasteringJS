Multipart MIME Types

Back to Overview

1. multipart/form-data

What it represents: Form data that includes files.

Use case: Used for forms that include file uploads. 

Content-Type: multipart/form-data

Example:

Content-Type: multipart/form-data; boundary=----WebKitFormBoundary7MA4YWxkTrZu0gW

2. multipart/byteranges

What it represents: Byte ranges of a single resource.

Use case: Used for sending partial content of a resource. 

Content-Type: multipart/byteranges

Example:

Content-Type: multipart/byteranges; boundary=000000000000000000

3. multipart/mixed

What it represents: A mix of different types of data.

Use case: Used for sending multiple parts of different types in a single message. 

Content-Type: multipart/mixed

Example:

Content-Type: multipart/mixed; boundary=frontier

4. multipart/alternative

What it represents: Different versions of the same content.

Use case: Used for sending different versions of the same content, such as plain text and HTML. 

Content-Type: multipart/alternative

Example:

Content-Type: multipart/alternative; boundary=boundary42

5. multipart/related

What it represents: Related data, such as an HTML file and its images.

Use case: Used for sending related resources together, such as an HTML file and its images. 

Content-Type: multipart/related

Example:

Content-Type: multipart/related; boundary=example-1
